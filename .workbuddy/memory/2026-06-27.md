### Bug：怪物（Goomba/Koopa）静止不动
- **根因**：Phaser Arcade Physics Group 的 `createCallbackHandler` 在 `group.add(sprite)` 时会遍历 `this.defaults`
  对 body 应用默认值，其中 `setVelocityX` 默认为 0（见 `node_modules/phaser/src/physics/arcade/PhysicsGroup.js`）。
  Goomba/Kooba 在构造函数中 `setVelocityX(-speed)` 设置的初始速度，紧接着被 `this.goombas.add(goomba)` /
  `this.koopas.add(koopa)` 重置为 0。而旧 `update()` 只在撞墙（blocked/touching）时才重设速度，
  没撞墙时什么都不做 → 速度恒为 0 → 怪物不动。
- **对比**：Fireball 正常，因为 `fireFireball()` 在 `this.fireballs.add(fireball)` **之后**才调用 `setVelocityX`。
- **修复**：在 Goomba/Koopa 的 `update()` 中每帧持续 `setVelocityX(direction * speed)`（Goomba 新增 `direction`
  属性初值 -1；Koopa 复用已有 `direction`）。即使 velocity 被组重置，下一帧立即恢复。
  - 改动文件：`src/entities/Goomba.ts`、`src/entities/Koopa.ts`
- **验证**：已通过 Phaser 源码确认 createCallbackHandler 逻辑；dev server 运行正常（HTTP 200）；
  `tsc --noEmit` 仅有项目预先存在的类型告警，未引入新错误。

### 备注
- Boss 未加入任何 physics group，velocity 不被重置，不受此 bug 影响，未改动。
- Mushroom 的 velocity 在 tween 完成（add 之后）才设置，也不受影响。

## 怪物悬崖检测（防止掉坑）

### 需求
让 Goomba/Koopa 在地面上来回巡逻，走到平台边缘时自动转身，不要走进坑里自杀。

### 实现
- 在 `Goomba` 和 `Koopa` 中各新增私有方法 `checkGroundAhead()`：
  - 沿当前移动方向，在怪物前缘（`body.width / 2 + 4`）和底部稍下方（`body.bottom + 4`）取一个探测点
  - 遍历 `this.scene.physics.world.staticBodies` 中的静态体，判断探测点是否落在某个静态体边界框内
  - 若没有任何静态体覆盖，说明前方是悬崖
- 在 `update()` 的左右碰撞转身逻辑之后，增加悬崖判断：
  - 当怪物着地（`body.blocked.down || body.touching.down`）且前方无地面时，立即 `direction *= -1` 转身
- Koopa 的两种状态都加了检测：
  - 普通行走状态（`!isShell`）
  - 被踢飞后的壳状态（`isShell && isMoving`）

## 宫殿触发范围与无限跳跃修复

### 问题
1. **刚走到宫殿边就卡住，约3秒后显示通关**：`checkEnterPalace()` 的触发范围太宽（`palaceX - 16` 到 `palaceX + palace.width/2 + 16`），玩家刚到宫殿左侧就被判定进入，随即 `physics.pause()` 暂停，玩家动不了。

### 修复

   - 新增 `enteringPalace` 标志，防止重复触发。
   - 根据 palace 纹理（160x200）中门的位置（x=68..92，y=144..200），把检测点精确放到门中心 `palaceX + 80`。

2. **跳跃控制**（`src/entities/Player.ts`）：

   - 松开跳跃键时立即重置 `canJump`、`jumpHeld` 和 `jumpHeldTime`。
   - 这样实现马里奥式的可变高度跳跃，按住跳得更高，但不会无限飞。

## 道具拾取与静止龟壳推开修复（第二轮）

### 问题
1. **金币、蘑菇、火球花还是吃不到**：上次的修复没触及根因。
2. **侧面碰到被踩扁的静止龟壳仍会死，不会推走**：`hitEnemy()` 虽写了推开分支，但推开瞬间龟壳变为 moving shell，同一帧/下一帧 overlap 再次触发，isMoving=true 导致玩家受伤。

### 根因分析
- **道具**：Phaser `PhysicsGroup` 的 `createCallbackHandler` 在 `group.add(child)` 时会遍历 `this.defaults`，其中 `setAllowGravity` 默认值为 `true`。Coin / FireFlower 在构造函数里把 `allowGravity` 设为 `false`，但紧接着 `this.coins.add(coin)` / `this.fireFlowers.add(flower)` 又把它重置回 `true`，导致金币、火球花受重力下落甚至消失。蘑菇加入组后 `velocityX` 被重置为 0，而 `Mushroom.update()` 只在撞墙时改速度，导致蘑菇常常静止不动。
- **龟壳**：推开静止龟壳时调用 `stomp()` 使其 `isMoving=true`，但玩家尚未离开龟壳碰撞范围，overlap 持续触发，第二次进入 `hitEnemy()` 时按 moving shell 判定为受伤。

### 修复
1. **道具组默认配置**（`src/scenes/GameScene.ts`）：
   - `this.coins = this.physics.add.group({ allowGravity: false });`
   - `this.fireFlowers = this.physics.add.group({ allowGravity: false });`
   - `this.mushrooms = this.physics.add.group({ allowGravity: true });`（与默认一致，显式声明便于维护）

2. **蘑菇持续移动**（`src/entities/Mushroom.ts`）：
   - `update()` 中每帧持续 `setVelocityX(this.speed)`，并把撞墙方向转换直接改为 `this.speed = ±Math.abs(this.speed)`，避免被组重置后静止。

3. **静止龟壳推开**（`src/entities/Koopa.ts`、`src/scenes/GameScene.ts`）：
   - `Koopa` 新增 `kickCooldown` 与 `kick(direction)` 方法：被玩家从侧面踢飞时设置为 moving shell，并给 150ms 的“刚被踢飞”冷却，此期间不会伤到玩家。
   - `hitEnemy()` 的 non-stomp 分支：
     - 普通 Koopa 或自然滑动的龟壳（`isMoving && !kickCooldown`）→ 受伤
     - 静止龟壳 → 调用 `e.kick(pushDirection)`，玩家获得向前的水平冲量 `setVelocityX(pushDirection * 150)` 和轻微上弹，立即离开龟壳。

- dev server 仍在运行（HTTP 200），Vite HMR 自动热更新，无需重启服务。
