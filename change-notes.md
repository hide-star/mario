# 本次调整说明

## 修改内容

1. **龟壳（被踩扁的 Koopa 壳）不再受“避免掉坑”限制**
   - 文件：`src/entities/Koopa.ts`
   - 改动：在 `isShell && isMoving` 分支中移除平台边缘转身逻辑，龟壳现在会一直向前滑动，即使平台尽头是坑也会掉下去。
   - 普通行走状态的 Koopa 仍保留该限制，只在龟壳滑动状态下解除。

2. **金币、蘑菇、火球花位置整体下调 20 像素**
   - 文件：`src/scenes/GameScene.ts`
   - 改动：
     - 场景内静态金币的 Y 坐标由 `+26` 调整为 `+46`。
     - 问号砖中弹出的道具（金币、蘑菇、火球花）的生成基准线 `spawnY` 由 `b.y - TILE_SIZE` 调整为 `b.y - TILE_SIZE + 20`。
   - 效果：这些道具的最终悬停/停留位置比之前低 20 像素，跳跃时更容易顶到/收集。

## 验证

- `npm run dev` 可正常启动（Vite 热更新无阻塞）。
- 项目原有 TypeScript 类型错误与本次改动无关，未引入新的语法或类型错误。

# 第二次调整：跳跃高度与终点布局

## 修改内容

1. **玩家跳跃高度提升约 20 像素**
   - 文件：`src/game/config.ts`
   - 改动：`PLAYER_JUMP` 从 `-450` 调整为 `-480`。
   - 效果：按住跳跃键到最大蓄力时，最大跳跃高度比之前高约 20 像素，高处金币/问号砖更容易够到。

2. **旗杆后移，并在前方添加 9 级阶梯**
   - 文件：`src/scenes/GameScene.ts`
   - 改动：
     - 旗杆位置 `goalX` 从 `158 * TILE_SIZE` 后移到 `164 * TILE_SIZE`。
     - 宫殿位置 `palaceX` 从 `166 * TILE_SIZE` 后移到 `172 * TILE_SIZE`。
     - 在旗杆前（x=155..163）添加 9 级阶梯，每级比前一级高一格，通到顶部平台（y = groundY - 9*32）。
     - 在旗杆与宫殿下方铺设顶部平台，使旗杆和宫殿都坐落在高台上。
     - 旗杆高度提升到 340 像素，旗杆与宫殿都放置在顶部平台表面。
     - `reachGoal` 中旗子下滑目标改为顶部平台表面上方 32 像素。
     - `checkEnterPalace` / `enterPalace` 中门的 Y 基准改为 `this.palace.y - 36`，与宫殿底部（平台表面）对齐。
   - 效果：终点区域更像经典马里奥的阶梯→旗杆→宫殿布局。

## 验证

- `npm run dev` 可正常启动（Vite 热更新无阻塞）。
- 未引入新的语法或类型错误。
